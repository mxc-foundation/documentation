开发

停止其他终端标签页中存在的任何进程

检查您是否还没有留下任何运行的lpwan-app-server实例（即，如果您已经运行了UI 和 npm start ）。

更改为兼容版本 Node.js

既然Docker容器将使用 Node.js v10.16.3，请在本地机器上切换到该版本。 节点版本管理器 让它方便做：

nvm install v10.16.3 &&
nvm use v10.16.3

经常从功能分支应用最新版本到您的任务分支：

git checkout feature/MCL-117 &&
git pull --rebase origin feature/MCL-117
git checkout luke/MCL-118-page-network-servers
git rebase -i feature/MCL-117

注意：运行 git rebase -i feature/MCL-117 的替代办法是合并并解决冲突而不是 git merge feature/MCL-117

在线重新加载进行调试：

在 LPWAN应用服务器构建后从Docker 容器运行， 如果您只是去http://localhost:8080，那么您将不会获得在线重新加载支持。 为了启用调试和在线重新加载，额外运行如下(在Docker 容器之外)：

cd lpwan-app-server &&
cd ui/ &&
npm start

然后在您的浏览器中请打开：http://localhost:3000

现在当您进行更改时，它将自动刷新

Library要求

UI中使用的所有库(Library)都应该提供反应本地支持

数据库访问权限

尝试使用 PostgreSQL GUI 来轻松解决测试数据库中的问题

示例︰

  • 下载http://www.psequel.com/
  • 输入您的 .env 文件或自定义配置文件中的连接详细信息：configuration/lora-app-server.toml
